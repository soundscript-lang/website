# Contributing

Use pull requests against `main` for changes to the public site and docs.

Before opening a pull request:

- run `npm install`
- run `npm run typecheck`
- run `npm run lint:docs`
- run `npm run spellcheck`
- run `npm run build`
- run `npm run test:smoke`

For broad content or information architecture changes, open or reference an
issue first so docs ownership and launch scope stay aligned.
